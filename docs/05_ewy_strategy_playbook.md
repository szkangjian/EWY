## 1. 产品概要

| 项目 | 内容 |
| ---- | ---- |
| 全称 | iShares MSCI South Korea ETF |
| 交易所 | NYSE Arca |
| 日均成交额 | ~$3.9 亿 |
| Spread | ~$0.02 |
| 核心驱动 | 半导体 / AI 周期 + 地缘政治冲击 |
| 当前策略基础 | 强势 regime 下的短线均值回归 |

EWY 本质上更接近“高波动半导体 beta”，而不是稳定分红资产。它适合做短线反弹，不适合用“时间站在你这边”的思路去扛仓。

## 2. 策略架构

```text
┌─────────────────────────────────────────┐
│  主策略：IBS 均值回归（收盘信号）         │
│  → IBS < 0.2 且 Close > MA200 → 买入     │
│  → IBS > 0.8 → 卖出                     │
│  → 28 笔，71% 胜率，+33.3% 总收益         │
├─────────────────────────────────────────┤
│  辅助策略：盘中跌幅反弹（观察型）         │
│  → 盘中跌破前日收盘 -3.5% → 买入         │
│  → 反弹 +2.0% 或持有 5 天 → 卖出         │
│  → 当前样本内有效，但不宜当主策略         │
└─────────────────────────────────────────┘
```

两个策略可以并行监控，但当前执行优先级很明确：
- `IBS` 是主策略
- `盘中跌幅反弹` 是辅助策略，仓位和预期都应更保守

## 3. 主策略：IBS 均值回归

### 3.1 规则

`IBS = (Close - Low) / (High - Low)`

- 买入条件：`IBS < 0.2` 且 `Close > MA200`
- 卖出条件：`IBS > 0.8`
- 安全阀：最大持有 `10` 个交易日

### 3.2 执行方式

| 环节 | 做法 |
|------|------|
| 买入 | 收盘前 5 分钟确认当日 IBS 接近阈值，使用 `MOC` 或收盘前市价单 |
| 卖出 | 收盘前 5 分钟确认 IBS > 0.8，按收盘卖出 |
| 工具 | `python ewy_signal.py` |

### 3.3 当前回测表现

| 指标 | 数值 |
|------|------|
| 交易笔数 | 28 |
| 胜率 | 71% |
| 总收益 | +33.3% |
| 均收益/笔 | +1.19% |
| 最大单笔亏损 | -5.18% |
| 平均持有 | 3.25 天 |
| 退出方式 | 28/28 全部 IBS>0.8 正常退出 |

### 3.4 执行上的关键认知

- 不需要先判断“为什么跌”，IBS 本身已经编码了当日多空博弈
- 这条策略依赖趋势环境，`MA200` 不是装饰品
- 它不是 80% 胜率圣杯，允许出现正常回撤

## 4. 辅助策略：盘中跌幅反弹

### 4.1 规则

| 条件 | 要求 |
|------|------|
| 入场 | 盘中价格相对前日收盘跌破 `-3.5%` |
| 止盈 | 从买入价反弹 `+2.0%` |
| 时间止盈/止损 | 持有满 `5` 个交易日到期平仓 |
| 风控 | 熔断中不接新信号 |

### 4.2 当前定位

这条策略保留，是因为修正口径后它仍然有正收益；但它已经不再适合被描述成“稳定主策略”。

当前样本内相对更好的组合是：

| 指标 | 数值 |
|------|------|
| 参数 | `-3.5% / +2.0% / 5天 / 无止损` |
| 交易笔数 | 21 |
| 胜率 | 76% |
| 总收益 | +18.3% |
| 均收益/笔 | +0.87% |
| 最大亏损 | -6.28% |

### 4.3 执行原则

- 只保留 **单笔持仓**
- 不做“连续触发自动网格加仓”的假设
- 如果主策略和辅助策略同日都触发，允许并行，但辅助策略仓位更小

## 5. 仓位管理

### 5.1 建议分配

| 用途 | 建议 |
|------|------|
| IBS 策略 | 核心仓位 |
| 跌幅策略 | IBS 的 `1/2` 到 `2/3` 仓位 |
| 现金储备 | 至少保留 `30%` |

### 5.2 最大同时持仓

- IBS：`1` 笔
- 跌幅策略：`1` 笔
- 合计最大同时持仓：`2` 笔

### 5.3 杠杆

**不用杠杆。**

EWY 没有 STRC 那种“股息覆盖利息”的缓冲，波动率又高，杠杆会把本来还能承受的短线回撤变成资金管理事故。

## 6. 风控规则

### 6.1 策略级熔断

| 条件 | 动作 | 当前代码行为 |
|------|------|--------------|
| 连续 `3` 笔到期亏损 | 暂停新信号 | 已实现 |
| 收盘跌破 `MA200` | 暂停新信号 | 已实现 |
| 收盘重新站回 `MA200` 且亏损链断裂 | 可恢复 | 已实现 |

### 6.2 单笔止损

当前仓库默认不对这两条策略额外设置单笔固定止损，原因是：

- IBS 的退出逻辑本来就是收盘信号驱动
- 盘中跌幅反弹在样本里经常先继续走弱，再完成反弹

但这不等于“永远不能停手”。如果出现以下情况，应直接人工暂停并复盘：

- 连续出现预期外的大额跳空
- 市场结构明显从单次急跌反弹，变成持续趋势下跌
- EWY 的基本面驱动从 AI/半导体强势切换到系统性衰退

## 7. 月度检查

每月 1 日运行 `python ewy_risk_monitor.py` 后，至少检查：

| 指标 | 警戒信号 |
|------|---------|
| 韩国半导体出口 YoY | 连续 2 月下滑 |
| DRAM 价格趋势 | 连续 4 周走弱 |
| ISM PMI | 连续 2 月低于 47 |
| 铜价 vs MA200 | 跌破均线 |
| 美国 10Y-2Y 利差 | 倒挂后重新转正 |

任意两个以上指标同时明显转坏，暂停策略。

## 8. 自动化工具链

### 8.1 收盘后信号

```bash
python ewy_signal.py
```

输出：
- 当日 IBS 值
- IBS 买入/卖出信号
- 盘中跌幅策略的日线近似信号
- 当前持仓与熔断状态

### 8.2 盘中实时监控

```bash
python realtime_ewy.py
```

实时监控：
- 盘中是否跌破前日收盘 `-3.5%`
- 持仓是否反弹达到 `+2.0%`
- 临近收盘时的 IBS 预估

### 8.3 每日数据更新

```bash
python update_ewy_today.py
```

### 8.4 qbot 入口

```bash
python ewy_orchestrator.py
python ewy_intraday_monitor.py
```

这两个入口现在和主仓库研究口径一致，也统一使用美东常规交易时段数据。

## 9. 执行清单

### 每日

- [ ] 开盘前确认实时监控工具正常
- [ ] 盘中只关注 `-3.5%` 辅助信号，不主观追跌
- [ ] 收盘后运行 `python ewy_signal.py`
- [ ] 若有 IBS 信号，优先执行 IBS
- [ ] 有交易就写入 [执行日志](06_ewy_execution_log.md)

### 每月

- [ ] 运行 `python ewy_risk_monitor.py`
- [ ] 检查宏观与半导体周期指标
- [ ] 确认没有触发策略级熔断
- [ ] 复盘过去一个月的真实胜率和均收益

### 每半年

- [ ] 将实盘结果与 [03_ewy_strategy_research.md](03_ewy_strategy_research.md) 的回测基准对比
- [ ] 如果 IBS 明显衰减，先暂停，再讨论调参
- [ ] 如果只剩辅助策略有效，不要强行把辅助策略升级成主策略
